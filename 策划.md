# 《剪影师》基础版游戏策划书

## Unity 2D 实现版 / Shadough Demo 版

---

## 一、游戏概述

### 1. 游戏名称

《剪影师》

英文项目名：Shadough

### 2. 游戏类型

轻动作解谜 / 空间机制解谜 / 低文本叙事独立游戏

### 3. 推荐开发引擎

Unity 2D

### 4. 推荐技术方案

Unity 2D + URP 2D Renderer

游戏采用“视觉光影”和“玩法影子”分离的技术结构。

视觉光影负责画面表现，例如灯光、明暗、阴影氛围。

玩法影子负责交互逻辑，例如剪下、携带、贴出、碰撞、压住机关、打开锁、吸引敌人。

二者视觉上应尽量一致，但逻辑上不直接等同。玩家剪下的不是 Unity 实时渲染出的阴影贴图，而是一个独立的 ShadowInteractable 玩法对象。

### 5. 核心关键词

影子、剪裁、拼贴、光源、里表世界、空间重组、原形态、碰撞解谜

### 6. 一句话概念

玩家扮演一名“剪影师”，可以剪下场景中某一刻真实存在的影子形态，并将它原封不动贴到世界中，使影子以自身形状、大小、角度和碰撞属性参与场景，从而创造道路、压住机关、打开锁或吸引敌人。

### 7. 核心卖点

本作的核心不是“获得某种影子道具”，而是“剪下一块具体形状的影子”。

影子不会因为贴到某个区域就自动变成桥、手、钥匙或机关工具。
影子贴出后保持原本形态。
它能否发挥作用，取决于玩家如何摆放它，以及它自身的形状、碰撞体和交互属性。

---

## 二、游戏定位

### 1. 目标体验

玩家在游戏中应获得以下体验：

发现一个普通物体的影子其实可以被利用。
通过观察影子的形状，判断它可能产生的作用。
剪下某一刻的影子形态。
把影子贴到合适的位置和角度。
看到影子依靠自身碰撞关系改变场景。
通过自己的摆放思路解决问题，而不是寻找固定钥匙。

### 2. 操作复杂度

基础操作包括：

移动
跳跃
剪下影子
贴出影子
调整贴影位置
调整贴影方向
短暂查看里世界影子结构

基础版中暂时不强制加入复杂光源形变系统。光源改变影子长度与方向可以作为第二轮增强机制，在核心剪影、贴影、碰撞解谜流程稳定后再加入。

### 3. 玩法深度来源

游戏深度来自以下方面：

同一个影子可以因为形状、角度、位置不同而产生不同用途。
影子不是固定功能道具，而是可摆放的空间实体。
玩家需要判断影子是否够长、够宽、够稳、角度是否合适。
机关不直接识别影子类型，而是识别影子的属性与碰撞关系。
玩家一次只能携带有限数量的影子，需要选择什么时候剪、在哪里贴。

### 4. Unity 2D 定位说明

Unity 负责实现：

2D 场景搭建
角色移动
剪影检测
自由贴影
碰撞检测
机关交互
实时 2D 光照
UI 提示
Demo 打包展示

其中 URP 2D Light 和 Shadow Caster 2D 主要服务于视觉表现。真正的玩法影子由自定义脚本系统管理。

## 三、世界观基础设定

### 1. 表世界

表世界是玩家正常活动的现实空间，拥有建筑、物体、机关、敌人和光源。

表世界中的物体大多无法被直接搬走，但它们在光照下会形成影子。剪影师无法改变物体本身，却可以通过改变影子间接改变场景结构。

### 2. 里世界

里世界不是另一张完全独立的地图，而是表世界背后的影子结构层。

玩家可以短暂进入“揭影视角”，看到隐藏的影子纹路、可剪影子、影子之间的连接关系，以及普通视觉下看不到的机关提示。

### 3. 剪影师

剪影师能够剪下、携带和贴出影子。他们剪下的不是“物体能力”，而是“某一刻的影子形态”。

剪影师的能力本质是重组世界的阴面结构。

### 4. 游戏主题

游戏主题不是恐怖，而是“重组世界的另一面”。

现实世界看似固定，但影子提供了另一种可塑空间。玩家通过剪影和贴影发现隐藏结构，并用影子创造新的路径。

## 四、影子系统重新定义

本作中的“影子”分为四个层级，必须清晰分离。

---

### 1. 视觉阴影 Visual Shadow

视觉阴影是 Unity 光照系统产生的画面效果。

它负责：

营造氛围
表现灯光方向
提示哪里可能存在影子
增强世界可信度

视觉阴影不直接参与玩法判定。

例如，Unity 2D Light 照到树后，地面出现树的阴影。这一层阴影只是视觉表现，不是玩家直接剪下的对象。

---

### 2. 可剪影子 Cuttable Shadow

可剪影子是玩家真正能剪下来的玩法对象。

它通常与视觉阴影位置接近，看起来像真实阴影，但逻辑上是独立对象。

在 Unity 中，可剪影子对应 ShadowInteractable。

它需要记录：

影子来源
影子显示图形
当前大小
当前旋转
碰撞体大小
碰撞体偏移
能否站立
能否压机关
能否开锁
能否吸引敌人
能否阻挡

可剪影子的重点不是“它属于哪种功能类别”，而是“它当前是什么形状，具有什么属性”。

---

### 3. 携带影子 Carried Shadow

玩家剪下影子后，影子进入玩家携带状态。

携带影子保存的是剪下那一刻的完整数据，而不是只保存一个类型名称。

例如玩家剪下树影时，系统保存：

来源是 Tree
当前是长条形
当前尺寸是多少
当前碰撞体多大
当前能否站立
当前能否压机关
当前能否吸引敌人

这意味着玩家不是获得“Tree 道具”，而是获得“一块具体形态的树影”。

---

### 4. 贴出影子 Pasted Shadow

贴出影子是玩家把携带影子放回世界后的实体对象。

贴出后必须保持剪下时的形态，不会自动变形成桥、手、钥匙或机关。

它能不能产生作用，取决于：

形状
大小
角度
位置
碰撞体
交互属性

例如：

树影如果足够长，并且被摆在裂谷上，就能成为临时道路。
树影如果碰到压力板，并且具备 canPress 属性，就能压住压力板。
钥匙影如果进入锁孔检测区，并且具备 canUnlock 属性，就能尝试开锁。
玩家影子如果具备 canAttractEnemy 属性，就能吸引寻影兽。

---

## 五、影子数据分类

本作不再使用“结构影 / 动作影 / 性质影 / 生物影”作为核心分类。

新的影子分类方式分为四个维度：

来源 Source
形态 Shape
属性 Properties
状态 State

---

### 1. 来源 Source

来源说明影子来自什么物体或角色。

例如：

Tree：树影
Beam：梁影
Key：钥匙影
Player：玩家影子
Clock：钟摆影
Monster：怪物影

来源只说明“影子从哪里来”，不直接决定功能。

错误理解：

Tree 等于桥。
Key 等于开门。
Player 等于诱饵。

正确理解：

Tree 只是树的影子。
它能不能当桥，要看它是否 canStandOn，以及形状是否真的覆盖裂谷。
Key 只是钥匙形影子。
它能不能开锁，要看它是否 canUnlock，以及是否进入锁孔检测区。

---

### 2. 形态 Shape

形态说明影子被剪下那一刻的真实样子。

包括：

长度
宽度
角度
缩放
Sprite
Collider Size
Collider Offset
是否细长
是否宽大
是否适合覆盖区域

同一个来源的影子，在不同形态下可能用途不同。

例如：

长树影可以跨越裂谷。
短树影不能跨越裂谷。
横向梁影可以压住压力板。
竖向梁影可能压不到压力板。
钥匙影方向不对时无法对准锁孔。

---

### 3. 属性 Properties

属性说明影子能与世界发生什么交互。

基础属性包括：

canStandOn：能否站立
canPress：能否压住机关
canUnlock：能否开锁
canAttractEnemy：能否吸引敌人
canBlock：能否阻挡角色或敌人

属性不是影子类型，而是交互能力。

示例配置：

TreeShadow：

canStandOn = true
canPress = true
canUnlock = false
canAttractEnemy = false
canBlock = true

KeyShadow：

canStandOn = false
canPress = false
canUnlock = true
canAttractEnemy = false
canBlock = false

PlayerShadow：

canStandOn = false
canPress = true
canUnlock = false
canAttractEnemy = true
canBlock = false

---

### 4. 状态 State

状态说明影子现在处于哪个阶段。

VisualShadow：视觉阴影，仅画面表现
CuttableShadow：可剪影子，场景中可被玩家剪下
CarriedShadow：携带影子，存放在玩家影子槽中
PastedShadow：贴出影子，已经放置到世界中参与交互
DecayingShadow：正在消散的影子

状态决定系统如何处理影子。

## 六、核心机制设计

---

### 1. 观察影子

玩家通过观察场景中的光影，发现可剪影子。

基础版中，光源变化可以暂时简化，先保证可剪影子的存在、识别、剪下和贴出流程成立。

后续增强版可以加入：

移动灯改变影子长度
移动灯改变影子方向
不同距离改变影子强度
多个光源产生多个候选影子

### Unity 2D 实现说明

视觉层使用 Unity 2D Light 和 Shadow Caster 2D。
玩法层使用 ShadowInteractable 表示可剪影子。
二者可以重合摆放，但不要混为同一个系统。

---

### 2. 剪影

当玩家靠近可剪影子时，可以按键将其剪下。

基础规则：

玩家初始一次只能携带一个影子。
剪下后，可剪影子从场景中隐藏或变淡。
玩家背包保存 ShadowItemData。
ShadowItemData 保存来源、形态和属性。
剪下影子失败时，应给出简单提示。

### Unity 2D 实现说明

ShadowCutter 负责检测最近的 ShadowInteractable。
ShadowInteractable 负责生成 ShadowItemData。
ShadowInventory 负责保存当前携带影子。
剪下后不只是保存 ShadowType，而是保存完整数据。

---

### 3. 贴影

玩家可以在自身一定范围内选择位置和方向，将携带影子贴出。

贴出后生成 PastedShadowObject。

贴影规则：

贴出的影子保持剪下时的原始形态。
贴出的影子不根据 ShadowType 自动变形。
贴出的影子不自动变成桥、手、钥匙或能力。
贴出的影子通过 Collider2D 与场景互动。
机关根据属性和碰撞关系响应，而不是根据来源标签硬匹配。

错误设计：

TreeShadow + 裂谷区域 = 自动生成桥
HandShadow + 拉杆区域 = 自动生成手
KeyShadow + 锁孔区域 = 自动开门

正确设计：

PastedShadowObject 保持原形。
如果它 canStandOn 且覆盖裂谷，玩家可以站上去。
如果它 canPress 且接触压力板，压力板被压住。
如果它 canUnlock 且进入锁孔区域，锁被打开。
如果它 canAttractEnemy，寻影兽会追逐它。

### Unity 2D 实现说明

FreeShadowPlacer 负责贴影位置、预览和放置。
PastedShadowObject 负责保存贴出影子的属性与碰撞体。
PastedShadowObject 不应根据 ShadowType 生成不同特殊效果。
所有机关优先读取 CanStandOn、CanPress、CanUnlock、CanAttractEnemy、CanBlock 等属性。

---

### 4. 揭影视角

玩家可以短暂进入揭影视角，观察隐藏影子结构。

在该状态下：

现实画面变暗。
可剪影子高亮。
影子轮廓变清晰。
隐藏机关浮现。
敌人与影子的关系被提示。

基础版中，揭影视角可以先作为视觉辅助，不承担复杂切换地图功能。

### Unity 2D 实现说明

可通过图层显隐、后处理、描边、高亮材质实现。
RevealViewController 控制可剪影子的高亮与隐藏提示。

---

## 七、机关设计原则

### 1. 裂谷 Gap

裂谷不识别 ShadowType。

它只需要物理上存在可站立平台。

任何贴出的影子，只要：

canStandOn = true
Collider2D 非 Trigger
位置和角度覆盖裂谷

玩家就可以站上去通过。

---

### 2. 压力板 Pressure Plate

压力板不应该只认某一种影子。

它检测：

是否有 PastedShadowObject 进入 Trigger
该影子 CanPress 是否为 true

如果满足条件，压力板激活，门打开。

这意味着树影、梁影、玩家影子都可能压住压力板，只要它们具备 canPress 属性并且摆放正确。

---

### 3. 锁孔 Lock

锁孔不直接判断 ShadowType == Key。

它检测：

是否有 PastedShadowObject 进入锁孔区域
该影子 CanUnlock 是否为 true
基础版可以先不检测角度
增强版可以要求角度接近正确方向

这样钥匙影不是“贴上自动开门”，而是一个具有开锁属性的形状对象。

---

### 4. 寻影兽 Shadow Seeker

寻影兽不追逐所有影子。

它追逐：

CanAttractEnemy = true 的 PastedShadowObject

玩家影子是基础版中最主要的诱饵影子。后续也可以加入其他能吸引敌人的特殊影子。

## 八、基础关卡设计

---

## 关卡一：树影过裂谷

### 场景内容

玩家面前有一道裂谷。裂谷旁边有树和树影。

### 目标

通过裂谷。

### 解法

玩家剪下树影。
在自身周围选择贴影位置和方向。
把树影贴到裂谷上。
如果树影足够长，并且位置与角度覆盖裂谷，玩家可以踩着它通过。

### 教学目的

让玩家理解：

影子可以被剪下。
影子贴出后保持原形。
影子不是自动变桥，而是依靠自身碰撞体成为临时道路。
摆放位置和角度会影响解法是否成功。

### Unity 2D 实现重点

TreeShadow 是 ShadowInteractable。
TreeShadow 的 canStandOn = true。
剪下后生成 ShadowItemData。
贴出后生成 PastedShadowObject。
PastedShadowObject 使用非 Trigger Collider2D。
玩家可以站在贴出的 TreeShadow 上。

---

## 关卡二：影子压板

### 场景内容

前方有一扇门，门旁边有压力板。附近存在可剪影子，例如梁影、树影或其他具有重量感的长条影子。

### 目标

打开门并通过。

### 解法

玩家剪下一个具备 canPress 属性的影子。
将影子贴到压力板上。
如果贴出的影子 Collider2D 覆盖压力板 Trigger，压力板被激活。
门打开，玩家通过。

### 教学目的

让玩家理解：

机关不一定需要特定影子类型。
只要影子具备对应属性，并且位置摆放正确，就能触发机关。
树影也可能压住压力板，梁影也可能压住压力板。
解谜重点是属性和碰撞关系，而不是固定钥匙匹配。

### Unity 2D 实现重点

PressurePlateController 控制压力板状态。
ShadowPressureTrigger 检测 PastedShadowObject。
如果 PastedShadowObject.CanPress = true，则激活压力板。
DoorController 根据压力板状态打开或关闭门。

---

## 关卡三：影钥匙

### 场景内容

前方有一扇上锁的门。场景中没有实体钥匙，但墙上有钥匙形装饰或钥匙形影子。

### 目标

打开门。

### 解法

玩家剪下钥匙形影子。
将它贴到锁孔检测区。
如果贴出的影子具备 CanUnlock 属性，并且进入锁孔区域，门打开。
增强版中可以要求钥匙影方向大致正确。

### 教学目的

让玩家理解：

影子的作用来自形状与属性。
钥匙影不是固定道具，而是一个具有 CanUnlock 属性的形状对象。
贴的位置和方向可以影响结果。

### Unity 2D 实现重点

KeyShadow 是 ShadowInteractable。
KeyShadow 的 canUnlock = true。
LockTrigger 检测 PastedShadowObject.CanUnlock。
基础版先只判断进入区域，后续可加入角度检测。

---

## 关卡四：寻影兽

### 场景内容

一个敌人守在通道前。它不会优先追逐玩家本体，而会被具有吸引属性的影子吸引。

### 目标

绕过敌人。

### 解法

玩家剪下自己的影子。
将自己的影子贴到远处墙面或地面。
贴出的玩家影子具有 CanAttractEnemy 属性。
寻影兽转而追逐该影子。
玩家趁机通过。

### 教学目的

让玩家理解：

自己的影子也是资源。
影子可以影响敌人行为。
影子不只是地形工具，也可以改变敌人的目标选择。

### Unity 2D 实现重点

PlayerShadow 生成 PastedShadowObject。
PlayerShadow 的 canAttractEnemy = true。
EnemyShadowSeeker 搜索 CanAttractEnemy 的 PastedShadowObject。
敌人追逐影子而不是直接追逐玩家。

---

## 九、敌人基础设计

### 1. 寻影兽

特点：

对影子敏感。
会追逐具有 CanAttractEnemy 的贴出影子。
不会被普通树影、钥匙影吸引，除非它们被配置为 canAttractEnemy。

玩法作用：

迫使玩家思考影子作为诱饵的用途。
增加空间压力。
强化“影子也是目标”的主题。

### Unity 2D 实现说明

寻影兽使用简单状态机：

巡逻
发现可吸引影子
追逐影子
攻击或停留
返回巡逻

它的目标不是固定 Player，而是 ShadowTarget 或具备 CanAttractEnemy 的 PastedShadowObject。

---

### 2. 噬影虫

特点：

会破坏玩家贴出的影子。
移动速度慢。
可用于制造时间压力。

玩法作用：

防止玩家无限依赖单一影子。
迫使玩家快速通过影桥或影子机关。

### Unity 2D 实现说明

噬影虫检测附近 PastedShadowObject。
接触后让影子进入消散或销毁状态。
基础版可以暂不实现，作为后续扩展。

---

### 3. 提灯人

特点：

携带强光。
强光会让影子变淡或加速消散。
按固定路线巡逻。

玩法作用：

让玩家注意光照危险。
形成轻度潜入玩法。
制造“需要光制造影子，但害怕强光破坏影子”的矛盾。

### Unity 2D 实现说明

提灯人使用 2D Light 表现强光。
强光范围使用 Trigger Collider。
PastedShadowObject 进入强光范围后加速消散。
基础版可以暂不实现。

---

## 十、基础限制规则

### 1. 携带限制

玩家初始只能携带一个影子。

这样可以迫使玩家做选择：

现在应该携带树影，还是钥匙影？
现在应该把影子用掉，还是保留给后面的机关？

### Unity 2D 实现说明

玩家拥有一个 ShadowInventory。
当前版本只保存一个 ShadowItemData。
ShadowSlot 已满时，不能继续剪影。

---

### 2. 时间限制

贴出的影子不是永久资源。

基础版可以先让贴出的影子永久存在，以便验证机制。
后续版本再加入持续时间与消散系统。

增强版规则：

普通影子持续时间中等。
结构型可站立影子持续时间略长。
玩家自己的影子持续时间较短。
强光区域会加速影子消散。

### Unity 2D 实现说明

后续可加入 ShadowDecay 脚本。
ShadowDecay 控制淡出、闪烁、销毁。
当前 Demo 优先保证剪影、贴影、碰撞解谜稳定。

---

### 3. 光照限制

影子在强光下会逐渐消散。

该机制属于第二轮增强内容。基础版可以先不加入，避免过早增加系统复杂度。

后续实现时，光照影响的是：

未剪下的可剪影子形态
携带影子的稳定性
贴出影子的消散速度

但贴出的影子不应随着灯光实时变形。
它应保持剪下那一刻的形态。

---

### 4. 代价机制

剪下影子可能影响原物体。

例如：

剪下树影后，树的影子短暂消失。
剪下自己的影子后，玩家短时间内更容易被里世界危险影响。
剪下怪物影后，怪物可能暂时失去部分能力，但会尝试找回影子。

基础版中可以先用“可剪影子隐藏 / 变淡 / 定时恢复”表现。

---

## 十二、基础版 Demo 设计

### Demo 场景名称

废弃钟楼

### 场景定位

这是一个小型半开放区域，用于展示《剪影师》的核心玩法。

### 场景元素

裂谷
树影
压力板门
钥匙影
上锁门
寻影兽
玩家影子
最终钟盘

### 玩家目标

进入钟楼顶层并启动中央钟盘。

### 可使用影子

树影
梁影或其他可压机关影子
钥匙影
自己的影子
钟摆影

### 基础流程

玩家进入钟楼一层，发现裂谷阻挡道路。
玩家剪下树影，将树影贴到裂谷上。
如果树影位置与角度合适，玩家踩着树影通过裂谷。

玩家来到压力板门前。
玩家剪下一个具备 canPress 的影子，并将其贴到压力板上。
压力板检测到影子后打开门。

玩家进入二层，发现上锁门。
玩家剪下钥匙形影子，将它贴到锁孔检测区。
锁孔检测 CanUnlock，门打开。

玩家进入钟楼内部，遇到寻影兽。
玩家剪下自己的影子，将其贴到远处作为诱饵。
寻影兽被 CanAttractEnemy 的影子吸引，玩家趁机通过。

玩家到达顶层，使用钟摆影或最终机关影子触发中央钟盘，完成 Demo。

### Demo 目标体验

让玩家在短流程中体验：

剪影
携带影子
自由贴影
原形态保持
用碰撞搭路
用属性压机关
用形状开锁
用影子影响敌人

### Unity 2D Demo 实现范围

Demo 使用一个 Unity Scene 完成。
场景采用灰盒平台或简单 2D Sprite 搭建。
光源使用 2D Light 作为氛围表现。
可剪影子使用 ShadowInteractable。
贴出影子使用 PastedShadowObject。
所有谜题通过碰撞体、Trigger 和简单状态机实现。

## 十三、美术风格

### 1. 整体风格

低饱和童话怪谈风格。

画面不走纯恐怖路线，而是偏神秘、旧纸、剪纸、暗色童话。

### 2. 表世界视觉

表世界颜色较暗，但保留现实质感。

推荐色彩：

灰蓝
米黄
旧木色
暗红
褪色白

### 3. 影子视觉

影子不能只是普通黑色块，应具有剪纸和墨迹质感。

可以表现为：

剪纸边缘
墨水流动
黑布褶皱
烟雾颗粒
裂纹轮廓
轻微抖动

### 4. 里世界视觉

进入揭影视角时：

现实物体褪色。
影子轮廓变清晰。
可剪影子出现边缘高亮。
隐藏机关以黑色纹路浮现。
被剪过的区域留下白色裂口。

### Unity 2D 美术实现说明

表世界使用 SpriteRenderer 或 Tilemap。
影子图层单独管理。
揭影视角可以通过后处理、透明遮罩、图层显隐和描边效果实现。

## 十五、Unity 2D 技术结构建议

### 1. 场景结构

建议 Unity 场景按以下结构组织：

Main Camera
Player
World
Lights
ShadowVisuals
ShadowLogic
Interactables
Enemies
UI
AudioManager
GameManager

### 2. 核心脚本模块

PlayerController
负责玩家移动、跳跃、基础物理。

ShadowCutter
负责检测可剪影子并执行剪影。

ShadowInventory
负责保存玩家当前携带的 ShadowItemData。

FreeShadowPlacer
负责贴影范围、贴影预览、贴影生成。

ShadowInteractable
负责场景中的可剪影子，并生成 ShadowItemData。

ShadowItemData
负责保存影子来源、形态和属性。

PastedShadowObject
负责贴出后的影子实体，提供 CanStandOn、CanPress、CanUnlock、CanAttractEnemy、CanBlock 等属性。

DoorController
负责门的开启和关闭。

PressurePlateController
负责压力板状态。

ShadowPressureTrigger
负责检测进入压力板的 PastedShadowObject。

LockTrigger
负责检测 CanUnlock 影子。

EnemyShadowSeeker
负责寻影兽追逐可吸引影子。

RevealViewController
负责揭影视角显示。

ShadowDecay
负责后续影子消散和强光影响。

### 3. 技术实现原则

视觉阴影和玩法影子分开处理。
ShadowType 只表示来源标签，不表示功能。
不要用 shadowType == Tree / Hand / Key 直接决定机关效果。
机关优先检测 PastedShadowObject 的属性。
贴出的影子必须保持剪下时的形态。
所有谜题优先通过碰撞体、Trigger、位置和角度表达。
Demo 优先保证机制清晰，而不是追求复杂系统。

## 十六、基础版开发范围建议

### 基础版只保留以下内容

一个主场景：废弃钟楼
一种基础移动角色
一个剪影系统
一个携带影子槽
一个自由贴影系统
树影过裂谷谜题
影子压板谜题
钥匙影开锁谜题
玩家影子诱敌谜题
一个最终机关
基础 UI 提示
简单胜利界面

### 暂不加入的内容

复杂技能树
大型开放世界
大量剧情文本
多角色系统
复杂战斗系统
复杂影子合成系统
多人模式
复杂光源实时形变
复杂存档系统

### 开发目标

基础版目标不是完成商业游戏，而是验证核心机制是否成立。

重点验证：

玩家是否能理解剪影和贴影。
影子保持原形是否直观。
玩家是否能通过摆放位置和角度解决问题。
机关检测属性而非类型是否更自由。
影子是否能自然产生多种用途。
玩法是否不依赖大量文本也能表达清楚。

## 十七、基础版核心总结

《剪影师》的核心不是“获得影子道具”，而是“剪下一块真实形态的影子”。

玩家看到影子。
玩家剪下影子。
玩家携带影子。
玩家贴出影子。
影子保持原形。
世界根据影子的形状、碰撞和属性发生变化。

最终体验可以总结为：

看见影子。
剪下影子。
保留形态。
重新摆放。
让影子以自己的形状改变世界。
